## ADDED Requirements

### Requirement: リクエスト名の保存時付与
リクエストの名前は編集フォームの常設フィールドとしてではなく、保存操作の時点で付与されなければならない(SHALL)。名前が未設定のまま保存しようとした場合、lazycurlは名前入力を促すプロンプトを表示し、入力された名前で保存を完了しなければならない(SHALL)。既に名前を持つリクエストを再保存する場合、名前入力プロンプトを表示してはならない(SHALL NOT)。

#### Scenario: 未命名リクエストの保存
- **WHEN** ユーザーが名前未設定のリクエストを編集中に`ctrl+s`を押す
- **THEN** 名前入力プロンプトが表示され、入力後にその名前でリクエストが保存される

#### Scenario: 既存リクエストの再保存
- **WHEN** ユーザーが既に名前を持つリクエストを編集し`ctrl+s`を押す
- **THEN** プロンプトは表示されず、既存の名前のままリクエストが保存される

#### Scenario: Adhocでの無名編集
- **WHEN** ユーザーがAdhocモードでリクエストを組み立てて送信する(保存しない)
- **THEN** 名前の入力を一切求められることなく編集・送信が完了する
