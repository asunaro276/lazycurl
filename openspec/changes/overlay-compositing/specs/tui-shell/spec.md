## ADDED Requirements

### Requirement: オーバーレイの合成描画
lazycurlは、help/environment選択/新規collection作成/保存先選択/リクエスト名入力/削除確認のいずれのオーバーレイを表示する場合も、背景の4パネル(Request/Response/Collections/History)を消さずに描画し続け、オーバーレイをその上に画面中央で重ねて表示しなければならない(SHALL)。ただし、オーバーレイの実際のレンダリングサイズが現在の端末サイズに収まらない場合は、そのオーバーレイに限り4パネルを描画せずオーバーレイのみを表示するフルスクリーン表示にフォールバックしなければならない(SHALL)。

#### Scenario: オーバーレイ表示中も背景パネルが見える
- **WHEN** ユーザーが`?`キーを押してヘルプオーバーレイを表示する(端末サイズがヘルプの表示に十分な場合)
- **THEN** 画面中央にヘルプの内容が表示され、その背景として4パネルの内容が引き続き表示される

#### Scenario: 端末が狭い場合のフルスクリーンフォールバック
- **WHEN** オーバーレイの実際のレンダリングサイズが現在の端末の幅または高さに収まらない
- **THEN** そのオーバーレイは4パネルを描画せず、オーバーレイの内容のみが画面全体に表示される

### Requirement: オーバーレイ表示中の背景パネルの減光表示
lazycurlは、オーバーレイを背景パネルに重ねて表示する間、背景の4パネルをオーバーレイ非表示時よりも彩度を落とした`dimmed`スタイルで描画しなければならない(SHALL)。

#### Scenario: オーバーレイ表示時の背景の見た目
- **WHEN** 背景の4パネルの上にオーバーレイが重ねて表示されている
- **THEN** 背景の4パネルはオーバーレイが表示されていないときよりも彩度の低い色合いで描画される
